# RLHF基础

RLHF（Reinforcement Learning from Human Feedback）是将人类偏好引入大语言模型训练的核心技术范式。与传统的监督微调（SFT）不同，RLHF 不再追求"模仿人类书写的正确答案"，而是学习"生成人类更偏好的结果"。这一范式的完整流程可分为两个紧密衔接的阶段。

阶段一：训练

Reward Model（奖励模型，简称 RM）是 RLHF 的"裁判系统"。其核心任务是为任意给定的（Prompt, Response）对输出一个标量奖励值，量化人类对该回复的偏好程度。

**模型架构设计**

RM 通常基于预训练语言模型（如 LLaMA、GPT 系列）进行改造。具体实现上，在模型的最后一层隐藏状态之上添加一个**回归头（Regression Head）**，将高维表示映射为单一标量值：

$$r_\theta(x, y) \in \mathbb{R}$$

其中 $x$ 表示输入提示（Prompt），$y$ 表示模型生成的回复（Response），$\theta$ 为 RM 的可训练参数。

架构选择上有两种主流方案：

* **共享参数方案**：复用预训练模型的全部参数，仅替换最后的输出层。优势在于保持强大的语义理解能力，劣势是训练成本较高。
* **冻结部分层方案**：冻结模型的底层参数（通常占 70%\~80%），仅微调顶层 Transformer 层和回归头。这能在保持 RM 判别能力的同时显著降低显存占用和训练时间。

**训练目标：Pairwise Ranking Loss**

RM 的训练数据由成对比较构成：对于同一 Prompt $x$，人类标注者从两个候选回复 $y\_w$（Win，被选中的）和 $y\_l$（Lose，被拒绝的）中选出更优者。训练目标是最大化以下似然函数：

$$\mathcal{L}_{\text{RM}}(\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( r_\theta(x, y_w) - r_\theta(x, y_l) \right) \right]$$

其中 $\sigma$ 为 Sigmoid 函数。该损失函数的本质是\*\* Bradley-Terry 模型\*\*的负对数似然，它假设两个物品被比较时的胜率可以用其潜在"强度"（此处即奖励值）的差值通过 Logistic 函数建模。

**关键超参数**

1. **Margin 设置**：实践中常引入显式的 Margin $\delta$ 来增强区分度： $$\mathcal{L} = -\log \sigma \left( r_\theta(x, y_w) - r_\theta(x, y_l) - \delta \right)$$ $\delta$ 通常设为 0.1\~0.5，防止模型对相近质量的回复给出过于接近的奖励值。
2. **In-batch Negative Sampling**：在一个训练批次（Batch）内，将其他样本的 $y\_l$ 作为当前样本的负例，可以在不增加标注成本的情况下扩大训练数据规模，提升 RM 的判别鲁棒性。

#### 1.1.2 阶段二：强化学习策略优化（PPO）

获得训练好的 RM 后，进入 RLHF 的第二阶段：使用强化学习算法（通常是 PPO，Proximal Policy Optimization）优化语言模型策略 $\pi\_\phi$，使其生成能获得更高 RM 分数的回复。

**优化目标**

$$\max_{\phi} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\phi(y|x)} \left[ r_\theta(x, y) \right] - \beta \mathbb{D}_{\text{KL}} \left[ \pi_\phi(y|x) \| \pi_{\text{ref}}(y|x) \right]$$

目标函数包含两个关键项：

* **奖励最大化项**：鼓励策略生成高 RM 分数的回复
* **KL 散度约束项**：防止策略 $\pi\_\phi$ 偏离参考模型 $\pi\_{\text{ref\}}$（通常是 SFT 模型）太远，维护语言模型的基础能力和输出分布稳定性

$\beta$ 为 KL 惩罚系数，是 RLHF 中最重要的超参数之一，通常需要在 0.01\~0.2 之间精细调优。

**Actor-Critic 架构**

PPO 算法采用经典的 Actor-Critic 结构：

* **Actor（策略模型 $\pi\_\phi$）**：即被训练的语言模型，负责生成回复。其参数通过策略梯度更新，朝着高奖励方向优化。
* **Critic（价值模型 $V\_\psi$）**：估计给定状态（已生成的 Token 序列）的期望累计回报，用于计算 Advantage（优势函数），降低梯度估计的方差。

Critic 通常从 RM 或 SFT 模型初始化，其输出头被改造为价值输出。价值模型的损失函数为： $$\mathcal{L}_{\text{critic}} = \left( R_t - V_\psi(x, y_{<t}) \right)^2$$

其中 $R\_t$ 为时刻 $t$ 的折扣回报。

**关键算法组件**

1. **Generalized Advantage Estimation（GAE）**：平衡偏差与方差的优势估计方法 $$\hat{A}_t = \sum_{l=0}^{\infty} (\gamma \lambda)^l \delta_{t+l}$$ 其中 $\delta\_t = r\_t + \gamma V(s\_{t+1}) - V(s\_t)$ 为 TD 误差，$\gamma$ 为折扣因子，$\lambda \in \[0,1]$ 控制偏差-方差权衡。$\lambda=1$ 时无偏但方差大，$\lambda=0$ 时有偏但方差小。
2. **Clipped Surrogate Objective**：PPO 的核心创新，限制单步策略更新幅度 $$L^{\text{CLIP}}(\phi) = \mathbb{E}_t \left[ \min \left( r_t(\phi) \hat{A}_t, \text{clip}(r_t(\phi), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$ 其中 $r\_t(\phi) = \frac{\pi\_\phi(a\_t|s\_t)}{\pi\_{\phi\_{\text{old\}}}(a\_t|s\_t)}$ 为概率比，$\epsilon$（通常 0.1\~0.2）为裁剪阈值。

### 1.2 偏好数据构造

RLHF 的效果上限直接受限于偏好数据的质量。与 SFT 的"正确答案"标注不同，偏好标注要求标注者在**成对比较**中做出相对判断，这引入了独特的统计学假设和工程挑战。

#### 1.2.1 Bradley-Terry 模型假设

Bradley-Terry 模型是成对比较数据分析的基石假设。该模型认为，每个候选回复 $y$ 具有一个隐含的"真实质量"参数 $\mu(y)$，当比较两个回复 $y\_i$ 和 $y\_j$ 时，$y\_i$ 被选中（胜选）的概率为：

$$P(y_i \succ y_j) = \frac{e^{\mu(y_i)}}{e^{\mu(y_i)} + e^{\mu(y_j)}} = \sigma(\mu(y_i) - \mu(y_j))$$

这一假设为 RM 的 Pairwise Ranking Loss 提供了理论基础。然而，该模型隐含了几个重要假设，在实践中常被违反：

1. **传递性假设**：若 $y\_1 \succ y\_2$ 且 $y\_2 \succ y\_3$，则应有 $y\_1 \succ y\_3$。但人类偏好常存在非传递性循环（Condorcet 悖论）。
2. **独立性假设**：不同比较对的标注相互独立。实际上，同一标注者的判断存在相关性。
3. **同质性假设**：所有标注者共享相同的偏好分布。现实中存在显著的个体偏好差异（如有的标注者偏好详细解释，有的偏好简洁回答）。

**工程实践建议**：

* 对标注者进行一致性筛选，计算 Kappa 系数或标注者间一致性（Inter-annotator Agreement），剔除偏差过大的标注者
* 引入个性化建模，允许 RM 学习不同人群的偏好分布
* 对违反传递性的"矛盾三元组"进行人工复核或自动剔除

#### 1.2.2 成对比较数据标注（Chosen vs Rejected）

**数据收集流程**

标准 RLHF 数据构造遵循以下 pipeline：

1. **Prompt 采样**：从真实用户查询或精心设计的指令集中随机采样 Prompt $x$。
2. **响应生成**：使用一个或多个"策略"（通常是不同阶段的 SFT 模型、不同规模的模型、或带不同采样的同一模型）生成 $N$ 个候选回复 ${y\_1, y\_2, ..., y\_N}$。
3. **成对组合**：从 $N$ 个回复中构造 $\binom{N}{2}$ 个比较对，或直接采样 $K$ 对（通常 $K \ll \binom{N}{2}$）。
4. **人工标注**：标注者查看 $(x, y\_i, y\_j)$，根据预定义的标注指南（Annotation Guideline）选择更优回复。
5. **质量控制**：引入"黄金标准"测试题（已知正确答案的隐藏测试对），过滤低质量标注者。

**标注指南设计**

高质量的标注指南是 RLHF 成功的关键。指南需明确界定评估维度，常见维度包括：

* **有用性（Helpfulness）**：回复是否直接解决了用户问题？
* **真实性（Honesty）**：回复是否包含事实错误或幻觉（Hallucination）？
* **无害性（Harmlessness）**：回复是否包含有毒、偏见或危险内容？
* **风格（Style）**：回复的清晰度、结构化程度、语气是否得当？

**标注格式演进**：

早期 RLHF（如 InstructGPT）采用**二元比较**（强制选择 Chosen/Rejected）。现代实践引入了**分级标注**：

* **Elo 评分制**：标注者不仅选出胜者，还需评估"胜利幅度"（如险胜、明显胜出、压倒性胜利）
* **多元评分**：对每个维度单独打分（1-7 Likert 量表），再综合为总体偏好

#### 1.2.3 Elo 评分体系在数据筛选中的应用

Elo 评分体系源自国际象棋排名系统，现被广泛应用于 RLHF 数据的质量评估和筛选。

**Elo 评分机制**

每个候选回复 $y$ 被赋予一个动态 Elo 分数 $R(y)$。当两个回复对战时，期望胜率为：

$$E_i = \frac{1}{1 + 10^{(R_j - R_i)/400}}$$

实际对战结果 $S\_i \in {1, 0, 0.5}$（胜、负、平）后，分数更新为：

$$R_i' = R_i + K \cdot (S_i - E_i)$$

其中 $K$ 为更新步长（通常 16\~32）。

**数据筛选应用**

1. **异常值检测**：Elo 分数分布的极端值（过高或过低）可能指示：
   * 标注错误（如明显错误的回复被错误标记为 Chosen）
   * 提示泄露（Prompt 包含答案线索）
   * 对抗性样本（故意构造的 trick 问题）
2. **数据去噪**：剔除 Elo 分数与多数人类判断不一致的样本。具体策略：
   * 计算每个样本的"共识度"：$C = \frac{\text{多数选择次数\}}{\text{总标注次数\}}$
   * 移除 $C < 0.6$ 的低共识样本（边缘案例保留用于训练鲁棒性，但高置信度错误应剔除）
3. **困难样本挖掘**：Elo 分数接近的样本对（即 $ |R\_i - R\_j| < \delta $）代表"难分难解"的比较，这些样本对 RM 训练价值最高，应优先纳入训练集。
4. **标注者能力评估**：通过每个标注者的判断与最终 Elo 排序的一致性，可以量化标注者能力，用于加权聚合或筛选。

**与 RM 训练的协同**

实践中常采用迭代式数据筛选：

* **第 0 轮**：使用简单启发式规则（如回复长度、格式规范性）构造初始训练集，训练初始 RM
* **第 1-n 轮**：使用当前 RM 为未标注样本预打分，结合 Elo 体系选择"高信息增益"样本（RM 不确定或人类判断分歧大的样本）进行标注
* **主动学习（Active Learning）**：优先标注 Elo 分数在决策边界附近的样本，最大化标注 ROI

这种 Elo-RM 协同筛选机制能将数据效率提升 30%\~50%，同时显著降低 RM 的分布外泛化误差。

### 一、为什么需要这些算法？（背景）

假设你有一个学生（模型），已经背完了数学公式（SFT预训练），但他解题时：

* 有时跳过步骤直接写答案（人类不喜欢，虽然可能对）
* 有时步骤混乱（人类看着累）
* 有时编造公式（幻觉）

你需要**一种训练机制**让他知道："人类喜欢什么样的回答格式？"

这就是 **RLHF（人类反馈强化学习）**，而PPO/DPO/GRPO是三种不同的实现路径。

***

### 二、PPO（Proximal Policy Optimization）—— **"私教+裁判"模式**

这是最经典但也最复杂的方案。

#### 核心角色（4个模型同时存在）

1. **Actor（学生/当前策略）**：正在学习的模型，负责生成答案
2. **Ref（参考答案/旧策略）**：冻结的旧版本，防止学生学歪（KL散度约束）
3. **Reward Model（裁判）**：独立的大模型，专门给回答打分（比如0-10分）
4. **Critic（价值教练）**：另一个大模型，预测"当前状态能得多少分"（类似围棋AI判断局势）

#### 训练流程（想象学生做数学题）

```
步骤1: 给学生一道题 → 学生生成答案（Actor采样）
步骤2: 裁判（Reward Model）给这个答案打分：7分
步骤3: 教练（Critic）说："根据我的经验，这题通常能拿8分，你这次7分，表现略差"
        → 计算优势（Advantage）= 实际7分 - 预期8分 = -1分
步骤4: 学生根据"优势"调整策略：下次要避免这次的做法
步骤5: 裁剪（Clip）机制：防止学生一次更新步子太大（比如从爱写步骤直接变成全跳步）
```

#### 关键公式拆解（别怕，看文字解释）

$$L^{CLIP}(\theta) = \mathbb{E}_t \left[ \min\left( r_t(\theta) \hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$

* $r\_t(\theta) = \frac{\pi\_\theta(a|s)}{\pi\_{old}(a|s)}$：**新策略生成这个动作的概率 / 旧策略生成这个动作的概率**
  * 如果 >1：新策略更倾向这个答案
  * 如果 <1：新策略更回避这个答案
* $\hat{A}\_t$：优势值（实际得分 - 预期得分）
* $\text{clip}$：裁剪，把$r\_t$限制在\[0.9, 1.1]之间，防止策略突变

**痛点**：需要同时加载4个大模型（Actor, Ref, Reward, Critic），**显存爆炸**。

### 三、DPO（Direct Preference Optimization）—— **"对比答案"模式**

PPO太麻烦了！DPO说：我们能不能**不用裁判打分，直接看人类标记的好/坏答案对比**？

#### 核心思想

人类已经准备了这样的数据：

* 问题：$x$ = "2+3=?"
* 好答案：$y\_w$ = "5"（步骤清晰）
* 坏答案：$y\_l$ = "步骤：2+3=6"（计算错误）

**目标**：让模型给好答案的概率尽量高，给坏答案的概率尽量低。

#### 直观公式

$$L_{DPO}(\theta) = -\mathbb{E}_{(x,y_w,y_l)} \left[ \log \sigma\left( \beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

拆解：

* $\pi\_\theta(y|x)$：**当前模型**生成答案$y$的概率
* $\pi\_{ref}(y|x)$：**旧模型**生成答案$y$的概率（防止走偏）
* $\frac{\pi\_\theta(y\_w|x)}{\pi\_{ref}(y\_w|x)}$：**当前模型比旧模型更喜欢好答案的倍数**
* **核心逻辑**：最大化 $\frac{\text{好答案的相对概率\}}{\text{坏答案的相对概率\}}$

**优势**：只需要2个模型（当前策略 + 参考策略），**显存友好**，训练简单。

**劣势**：依赖静态的偏好对数据，缺乏探索（不会主动尝试新回答风格）。

### 四、GRPO（Group Relative Policy Optimization）—— **"班级排名"模式**

这是DeepSeek提出的折中方案：**既想要PPO的在线探索能力，又受不了PPO的4模型开销**。

#### 核心创新：砍掉Critic，用"组内相对评分"代替绝对价值判断

#### 训练流程（关键区别在这里）

```
步骤1: 给同一道题，让模型生成G个不同答案（比如G=4）：
        - 答案A: 5（正确，步骤清晰）
        - 答案B: 5（正确，步骤啰嗦）
        - 答案C: 6（计算错误）
        - 答案D: 4（计算错误）

步骤2: 裁判（Reward Model或规则）给每个答案打分：
        - A: 9分, B: 7分, C: 2分, D: 1分

步骤3: 计算组内统计：
        - 组平均分 μ = (9+7+2+1)/4 = 4.75
        - 组标准差 σ = 3.4
        
步骤4: 计算每个答案的优势（Z-score标准化）：
        - A的优势 = (9-4.75)/3.4 = +1.25（比平均好）
        - B的优势 = (7-4.75)/3.4 = +0.66（比平均好）
        - C的优势 = (2-4.75)/3.4 = -0.81（比平均差）
        - D的优势 = (1-4.75)/3.4 = -1.11（比平均差）

步骤5: 用这些优势值，像PPO一样更新策略（有clip机制）
```

#### 为什么这样有效？

* **不需要Critic预测价值**：直接用**同一组内的相对比较**代替绝对价值估计
* **适合规则明确任务**：数学、代码题可以写规则自动打分（对就给1，错就给0），不需要训练昂贵的Reward Model
* **显存省一半**：只要3个模型（Policy, Ref, Reward），且Reward可以是轻量规则

#### 关键公式对比

**PPO的优势计算**：$\hat{A}_t = r\_t + \gamma V(s_{t+1}) - V(s\_t)$ （需要Critic网络$V$）

**GRPO的优势计算**：$A\_i = \frac{r\_i - \text{mean}({r\_1,...,r\_G})}{\text{std}({r\_1,...,r\_G})}$ （纯统计，无网络）

### 五、终极对比：三个维度

| 维度          | PPO                          | DPO            | GRPO                                |
| ----------- | ---------------------------- | -------------- | ----------------------------------- |
| **需要几个模型？** | 4个（Policy+Ref+Reward+Critic） | 2个（Policy+Ref） | 3个（Policy+Ref+Reward），但Reward可用规则代替 |
| **数据怎么来？**  | 在线生成，实时打分                    | 离线静态偏好对        | 在线生成，每组G个答案                         |
| **评分方式**    | 绝对分数（0-10分）+ Critic预测        | 相对比较（好>坏）      | 组内相对排名（Z-score）                     |
| **显存消耗**    | 🔴极高                         | 🟢低            | 🟡中（比PPO省很多）                        |
| **适合什么任务？** | 复杂开放域（需要精细价值判断）              | 主观偏好（风格、安全）    | 客观可验证任务（数学、代码）                      |
| **训练稳定性**   | 高（有Critic兜底）                 | 中（依赖数据质量）      | 高（相对排名抗干扰）                          |

### 六、一句话记忆法

* **PPO**：请4个专家（Actor, Ref, Reward, Critic）陪读，精准但贵
* **DPO**：直接看标准答案对比学习，简单但死板（不会举一反三）
* **GRPO**：让模型一次写4个答案，选其中好的相对奖励，**性价比之王**
