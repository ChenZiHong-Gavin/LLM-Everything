# 文本表示方法

#### 第 2 篇：经典文本表示方法——从词袋到哈希

**标题建议：** _TF-IDF、BM25、SimHash：文本变成向量的经典三条路_

**内容：**

* **词袋模型思想**：把文本看成词的集合，忽略顺序
* **TF-IDF**
  * TF（词频）和 IDF（逆文档频率）各自的直觉
  * 为什么 IDF 要取 log
  * TF-IDF 向量 + 余弦相似度 = 经典文本相似度方案
  * 局限：无法处理同义词、一词多义
* **BM25**
  * 从 TF-IDF 到 BM25 的演进：饱和函数 + 文档长度归一化
  * k1、b 两个超参数的含义与调参直觉
  * 为什么 BM25 至今仍是搜索引擎的标配召回算法
  * BM25 vs TF-IDF 的效果差异
* **SimHash**
  * 局部敏感哈希（LSH）的思想：相似的输入 → 相似的哈希值
  * SimHash 的构造过程（加权 → 合并 → 二值化）
  * 用汉明距离快速判断文本是否近似重复
  * 经典应用：搜索引擎网页去重
* **三种方法的定位对比**：精排 vs 召回 vs 去重
